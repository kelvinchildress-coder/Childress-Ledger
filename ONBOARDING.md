# The Family Ledger — Cowork Onboarding (v2)

**Read this file first.** Everything you need to be productive on this project lives here.

---

## Mission

A shared household task tracker for two parents (and configurable kids). One source of truth for everything from "pay mortgage" to "schedule dentist," visible to both parents on any device. Sunday-morning email + optional push summary, AI-assisted task entry, and a self-improving insights view that flags drift and patterns.

The product is called **The Family Ledger**. Visual aesthetic is "warm paper ledger" — cream background (#FAF7F2), navy ink (#1B2C3A), terracotta accent (#C9603C), serif display font (Fraunces) + clean sans body (DM Sans).

## Architecture in 60 seconds

```
                            ┌────────────────────┐
   ┌──────────────┐         │   Google Sheet     │
   │ PWA (this    │  HTTPS  │   - Tasks tab      │
   │ folder, in   │ ◄─────► │   - Settings tab   │
   │ browser/PWA) │         │   - PushSubs tab   │
   └──────┬───────┘         └─────────┬──────────┘
          │                           │ bound script
          │ IndexedDB                 ▼
          │ (offline cache)   ┌────────────────────┐
          │                   │  Apps Script        │
          ▼                   │  - doGet / doPost   │
   ┌──────────────┐           │  - sendWeeklyEmail  │
   │  Workbox     │  POST     │  - sendDailyDigest  │
   │  bg sync     │ ◄──────── │  - processReplies   │
   │  (24h queue) │           │  - aiProxy (Claude) │
   └──────────────┘           │  - buildIcs         │
                              └─────────┬──────────┘
                                        │
              ┌─────────────────────────┼──────────────────────┐
              ▼                         ▼                      ▼
      ┌───────────────┐         ┌───────────────┐    ┌────────────────────┐
      │ Anthropic API │         │ Parents' inbox│    │ Cloudflare Worker  │
      │ (AI agent)    │         │ (Sun 7am)     │    │ (Web Push relay)   │
      └───────────────┘         └───────────────┘    └────────────────────┘
```

**Data flow:**

1. User changes something in the UI → React state updates → `persist()` is called.
2. `persist()` writes to IndexedDB immediately (offline-safe).
3. `persist()` debounces 800ms, then POSTs to Apps Script `?action=save`. If offline, Workbox queues the POST and replays when network returns.
4. Apps Script writes to the Tasks tab and stamps `lastModified`.
5. Every 60s the app calls `?action=load` and checks for tasks whose `lastModified` is newer than ours. If any differ from the local copy, the conflict banner appears with adopt/keep choices.
6. On next app load, Apps Script returns the Sheet contents (Sheet is source of truth on cold start).
7. Local IndexedDB is the fallback when offline.

**Last-write-wins** on concurrent edits, but the conflict banner makes silent overwrites visible.

## Design decisions worth knowing (don't undo these)

- **Single-file App.jsx.** Still ~1800 lines in one file. The new modules (sync/identity/ai/insights/push) split out cleanly because they were genuinely orthogonal to UI. The UI itself stays in App.jsx.
- **`window.storage` → `storage.js` wrapper.** Untouched. Same shape.
- **Apps Script POST uses `Content-Type: text/plain`.** Still intentional — avoids the CORS preflight. Don't "fix" to `application/json`.
- **No explicit `mode: "cors"` on fetches anymore.** It was tripping up the redirect from `script.google.com` → `script.googleusercontent.com`. Letting the browser default solved the original test-connection bug.
- **AI key lives in Apps Script Script Properties, never in the browser.** This is the recommended path because (a) the key never sits in IndexedDB, (b) one config covers every device.
- **Debounced sync at 800ms.** Same as before.
- **Local IndexedDB AND remote sheet.** Same as before. Don't drop the local cache.
- **Categories and colors are coupled.** Same as before.
- **Streak calculation handles two cases.** Same. Plus there's now `personalDailyStreak()` in `identity.js` which scopes to one person's completions.
- **Migration on load.** `migrate()` now also fills in `completionLog`, `lastModified`, and `createdAt`. Old `completionHistory`-only tasks attribute to `"Family"` so the leaderboard doesn't crash.

## Gotchas

- **The Apps Script Web App URL is effectively a password.** Anyone with it can read/write the Sheet. The optional `SHARED_SECRET` adds defense-in-depth.
- **iOS PWAs have specific quirks.** Push requires iOS 16.4+, the PWA must be installed first (not just bookmarked), and Safari is the only entry point.
- **The Apps Script trigger runs as the script owner's identity**, not the user's. Email comes from whichever Google account set up the script.
- **Gmail label-based reply parsing has a ~10 minute delay** (the trigger interval). For "instant," wire up Pub/Sub watch — not done because the file count went up enough already.
- **`Browser.msgBox()` in `setupAll`** is wrapped in try/catch — it only works when run manually from the editor.
- **The `.ics` export uses UTC timestamps with 9am local time as default.** Calendar apps adjust. The new `webcal://` feed is preferred — it stays current automatically.
- **Push relay is Cloudflare-specific in the supplied scaffold.** You can use any service that can perform ECDH/HKDF; Apps Script can't do the crypto itself, so we proxy.
- **Anthropic rate limit in Apps Script:** the proxy enforces 60 calls/hour per script. If you blow through that, the UI surfaces "Rate limited — try again in Ns."

## Self-improvement loop

The ledger learns and adapts in two layers:

**Local heuristics (always on, no AI required):**
- `insights.js` watches a rolling 1000-event log (completes, snoozes, deletes, quick-adds).
- The quick-add category and frequency dropdowns reorder by 30-day-decayed usage — your most-used categories float to the top.
- Stale tasks (no completion in 30+ days) get listed in Insights for cleanup.
- Snooze patterns (you always push HVAC by exactly 7 days, 3 times in a row) get flagged with "consider changing the frequency."
- Quick-adds typed 3+ times with the same title across 14+ days get a "looks recurring" suggestion.

**AI layer (opt-in, requires backend + `ANTHROPIC_API_KEY`):**
- **Quick-add AI Fill:** Claude reads the title and proposes category/frequency/priority/assignee/deadline.
- **Deadline parser:** "next Friday" / "end of month" → ISO date.
- **Weekly retrospective:** Run-once button in Insights. Claude summarises wins/drift and suggests one concrete change.

Both layers are durable: each event is appended to `family_ledger_events_v1` in IndexedDB. You can rip out the AI layer at any time without affecting the local heuristics.

## How to verify changes

**Local dev:**

```bash
npm install
npm run dev          # localhost:5173 with HMR
```

**Production build sanity:**

```bash
npm run build
npm run preview      # localhost:4173 — needed to exercise the SW
```

**PWA install test:**
DevTools → Application → Service Workers (registered) → Manifest (icons + colors). For mobile, deploy to Vercel and open on a real device.

**Backend connection test:**
Settings → paste URL → "Test connection" should return ✓ Connected within ~2s. If it fails, the inline banner now shows the specific error reason and hint. "Copy debug info" gives a JSON blob.

**Sync test:**
1. Add a task → row appears in the Sheet within ~1s.
2. Edit a cell directly in the Sheet (change something's title) and bump its `lastModified` column to `Date.now()`.
3. Reload the app within 60s → conflict banner appears.

**AI test:**
Quick-add: type "schedule dentist" → click **AI Fill**. Should drop category=Kids Activities, frequency=monthly within ~2s.

**Push test (after relay setup):**
Settings → Enable push. Trigger `sendWeeklyEmail` manually from the Apps Script editor — your phone should ping.

## What I wouldn't change without asking

- The aesthetic (cream paper + navy + terracotta).
- The "Family Ledger" naming.
- The single-file App.jsx structure.
- The 10 specific categories.
- The reply command syntax (`ADD: title · category · frequency · assignee`). Outgoing emails reference it verbatim.
- The "AI proxied through Apps Script" decision. Pushing the key to the browser is a footgun.

**Fair game to refactor freely:**
- Internal helper functions in App.jsx if you find a cleaner abstraction.
- Style values (within the established palette).
- Apps Script internal functions, as long as `doGet`/`doPost`/`sendWeeklyEmail`/`processReplies`/`aiProxy_`/`buildIcs_` keep their contracts.
- The Insights view — it's the most actively-evolving area. Adding new heuristics is cheap and low-risk.

## Quick context for a fresh session

The most common reasons a Claude agent picks this project back up:

1. **"Sync is failing."** Open the connection-test panel. The hint usually points you straight at the deployment-access setting. The "Copy debug info" button captures the actual error response.
2. **"I want to enable AI / push."** See `DEPLOYMENT.md`. The two are independent — AI needs `ANTHROPIC_API_KEY` in Script Properties; push needs the Cloudflare Worker + VAPID keys.
3. **"Add another feature."** Most additions belong in App.jsx or a new module under `src/`. The `Insights` view is the natural home for any new heuristic.
